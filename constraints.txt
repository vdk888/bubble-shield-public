# Bubble Shield installer dependency pins.

# These versions EXACTLY match the wheels vendored in vendor/wheels/. They are
# passed to `pip install -c constraints.txt` by install-app.sh on BOTH the
# offline path (vendored wheels) and the online PyPI fallback path, so the two
# resolve to identical versions.
#
# Why this file exists: the online fallback used to be unpinned and would pull
# the LATEST release from PyPI (e.g. pywebview 6.x) against launcher code that
# targets pywebview 3.4 — the exact API-drift crash class (window.events
# AttributeError) that shipped to a client. Pinning here keeps the fallback
# byte-for-version identical to the offline install.
#
# Maintenance: when vendor/wheels/ is re-staged with new versions, regenerate
# this file from the wheel filenames so it stays in lockstep.
